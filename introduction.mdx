Alita Robot is a modern Telegram group management bot built with Go and the [Gotgbot](https://github.com/PaulSonOfLars/gotgbot) library. It gives administrators comprehensive tools to moderate communities of any size — banning and muting users, filtering spam, welcoming new members, managing notes, and more — all backed by PostgreSQL persistence and Redis caching for Go-native speed. This page covers what Alita is, its key capabilities, and how its architecture fits together.

## What Alita does

Alita runs inside Telegram groups as an admin bot. Once you add it to a group and promote it to admin, it responds to commands from group admins and monitors messages according to the rules you configure. It can:

- **Moderate users** — ban, mute, kick, warn, promote, and demote with configurable warn limits
- **Filter content** — catch keyword matches with regex support and auto-respond or delete
- **Block spam** — flood control, word and sticker blacklists, CAPTCHA verification for new members
- **Greet members** — fully customizable welcome and goodbye messages with variable substitution and media attachments
- **Save notes** — store formatted messages and retrieve them on demand with `/get`
- **Manage pins** — pin and unpin messages with anti-spam protection
- **Lock message types** — prevent links, forwards, media, and other content types from being posted

## Architecture overview

Alita is a single Go binary that connects to two external services:

| Component | Role |
|-----------|------|
| **Go + Gotgbot** | Core bot runtime; handles Telegram updates via polling or webhooks |
| **PostgreSQL** | Persistent storage for group settings, user records, filters, notes, and warnings |
| **Redis** | Two-tier cache with TTL and stampede protection; reduces database load |

The bot uses a worker-pool architecture for concurrent update processing. A configurable dispatcher handles up to 200 goroutines by default, with separate worker pools for database operations, cache operations, bulk processing, and background statistics collection. A four-layer panic recovery system (dispatcher → worker pool → decorator → handler) ensures a single bad update never crashes the bot.

Updates arrive either through long polling (default, zero infrastructure needed) or webhooks (recommended for production, supports Cloudflare Tunnel).

## Supported languages

Alita ships with built-in translations for the following locales. Set `ENABLED_LOCALES` in your `.env` to activate them:

| Code | Language |
|------|----------|
| `en` | English (default) |
| `es` | Spanish |
| `fr` | French |
| `hi` | Hindi |
| `ru` | Russian |
| `pt` | Portuguese |
| `id` | Indonesian |
