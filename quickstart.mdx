This guide walks you through deploying Alita Robot using Docker Compose — the fastest path from zero to a running bot. Docker Compose brings up the bot, a PostgreSQL database, and a Redis instance together, so you don't need to provision any external services. By the end you'll have Alita running in a Telegram group and responding to commands.

## Prerequisites

Before you begin, make sure you have:

- **Docker and Docker Compose** installed on your machine or server
- **A Telegram Bot Token** — create a bot with [@BotFather](https://t.me/BotFather) and copy the token it gives you
- **Your Telegram user ID** — send a message to [@userinfobot](https://t.me/userinfobot) to find it
- **A Telegram channel or group for logs** — Alita writes admin actions and errors to a log channel; note its ID (starts with `-100`)

## Setup steps

<Steps>

<Step title="Clone the repository">

Clone the Alita Robot source and move into the project directory:

```bash
git clone https://github.com/divkix/Alita_Robot.git
cd Alita_Robot
```

</Step>

<Step title="Create your .env file">

Copy the sample environment file to `.env`:

```bash
cp sample.env .env
```

Open `.env` in your editor and fill in the required values:

```env
# Required — get your bot token from @BotFather
BOT_TOKEN=123456:ABC-DEFghijklmnopqrstuvwxyz

# Required — your Telegram user ID (numeric)
OWNER_ID=123456789

# Required — log channel ID (must start with -100)
MESSAGE_DUMP=-1001234567890

# Database — these match the defaults in docker-compose.yml
DATABASE_URL=postgres://postgres:password@postgres:5432/alita_robot?sslmode=disable

# Redis — matches the Redis service in docker-compose.yml
REDIS_ADDRESS=redis:6379
REDIS_PASSWORD=
```

<Note>
  `AUTO_MIGRATE=true` is set automatically inside the Docker Compose configuration, so database tables are created on first startup without any extra steps.
</Note>

</Step>

<Step title="Start all services">

Run Docker Compose in detached mode to start Alita, PostgreSQL, and Redis:

```bash
docker-compose up -d
```

You should see output similar to:

```
[+] Running 4/4
 ✔ Network alita_robot_default     Created
 ✔ Container alita-postgres        Started
 ✔ Container alita-redis           Started
 ✔ Container alita-bot             Started
```

</Step>

<Step title="Verify the bot started successfully">

Check the bot's logs to confirm it connected to Telegram and finished loading modules:

```bash
docker-compose logs -f alita
```

Look for lines like:

```
{"level":"info","msg":"[Config] Configuration loaded and validated successfully"}
{"level":"info","msg":"Bot started in polling mode"}
{"level":"info","msg":"All modules loaded successfully"}
```

Press `Ctrl+C` to stop following the logs. The bot keeps running in the background.

</Step>

<Step title="Add the bot to a group and promote it to admin">

1. Open Telegram and create a new group, or open an existing one you manage.
2. Add your bot by searching for its username (the one you set with @BotFather).
3. Open the group's member list, tap the bot's name, and select **Promote to admin**.
4. Grant it the permissions it needs: **Delete messages**, **Ban users**, **Add new admins**, and **Pin messages**.

</Step>

<Step title="Test the bot">

Send `/start` or `/ping` in the group. Alita should respond immediately.

Try a few admin commands to confirm everything is working:

```
/id         — returns the chat ID and your user ID
/ping       — checks bot response time
/help       — lists all available commands
```

</Step>

</Steps>

## Managing your deployment

Common Docker Compose commands for day-to-day operation:

```bash
# View live logs
docker-compose logs -f alita

# Stop all services
docker-compose down

# Restart the bot after changing .env
docker-compose up -d --force-recreate alita

# Pull the latest image and restart
docker-compose pull && docker-compose up -d
```
